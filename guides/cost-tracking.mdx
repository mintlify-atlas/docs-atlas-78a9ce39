## Overview

GSD tracks every token and dollar spent during execution. The cost dashboard shows real-time breakdowns by phase, slice, model, and milestone.

## The Cost Dashboard

Press `Ctrl+Alt+G` or run:

```bash
/gsd status
```

The dashboard shows:

<CardGroup cols={2}>
  <Card title="Current Position" icon="map-pin">
    - Active milestone, slice, and task
    - Current phase (Research, Plan, Execute, Complete)
    - Elapsed time in auto mode
  </Card>
  
  <Card title="Cost Breakdown" icon="dollar-sign">
    - Per-unit cost (task, slice, milestone)
    - Token usage (input and output)
    - Cost by model (Opus vs Sonnet)
    - Running total for the milestone
  </Card>
  
  <Card title="Projections" icon="chart-line">
    - Estimated total cost based on completed work
    - Average cost per task/slice
    - Budget ceiling status
  </Card>
  
  <Card title="Progress" icon="list-check">
    - Completed units (tasks, slices)
    - In-progress units
    - Remaining work in the roadmap
  </Card>
</CardGroup>

## Per-Phase Model Selection

Use different models for different phases to optimize cost and quality.

### Model Strategy

- **Research** — Fast, cheap model (e.g., Sonnet, DeepSeek)
- **Planning** — High-quality model (e.g., Opus, o1)
- **Execution** — Balanced model (e.g., Sonnet)
- **Completion** — Fast model (e.g., Sonnet, Haiku)

### Configure in Preferences

<Steps>
  <Step title="Open preferences">
    ```bash
    /gsd prefs
    ```
  </Step>
  
  <Step title="Set models per phase">
    ```yaml
    ---
    version: 1
    models:
      research: claude-sonnet-4-6
      planning: claude-opus-4-6
      execution: claude-sonnet-4-6
      completion: claude-sonnet-4-6
    ---
    ```
    
    Use expensive models where quality matters (planning, complex execution) and cheaper models where speed matters (research, simple completions).
  </Step>
  
  <Step title="Save and continue">
    GSD picks up the new model selection at the next phase boundary. No need to restart.
  </Step>
</Steps>

### Example: Cost-Optimized Setup

```yaml
models:
  research: openrouter/deepseek/deepseek-r1  # $0.55/M tokens
  planning: claude-opus-4-6                   # $15/M input, $75/M output
  execution: claude-sonnet-4-6                # $3/M input, $15/M output
  completion: claude-sonnet-4-6               # $3/M input, $15/M output
```

This setup uses DeepSeek for fast research, Opus for high-stakes planning, and Sonnet for execution.

## Budget Ceilings

Set a budget ceiling to auto-pause when spending reaches a limit.

### Enable Budget Ceiling

<Steps>
  <Step title="Edit preferences">
    ```bash
    /gsd prefs
    ```
  </Step>
  
  <Step title="Set ceiling in USD">
    ```yaml
    budget_ceiling: 50.00
    ```
    
    When the milestone's total cost reaches $50, auto mode pauses.
  </Step>
  
  <Step title="Check status anytime">
    ```bash
    /gsd status
    ```
    
    The dashboard shows how close you are to the ceiling.
  </Step>
</Steps>

### What Happens When the Ceiling is Reached

1. Auto mode completes the current task
2. State is saved cleanly
3. A message displays: "Budget ceiling reached ($50.00)"
4. You can review progress and decide whether to continue

### Resume After Ceiling

<Steps>
  <Step title="Review spending">
    ```bash
    /gsd status
    ```
    
    Check the cost breakdown to see where tokens were spent.
  </Step>
  
  <Step title="Adjust ceiling if needed">
    ```bash
    /gsd prefs
    ```
    
    Increase the ceiling or remove it:
    
    ```yaml
    budget_ceiling: 100.00  # or omit to disable
    ```
  </Step>
  
  <Step title="Resume auto mode">
    ```bash
    /gsd auto
    ```
  </Step>
</Steps>

## Cost Projections

GSD estimates total milestone cost based on completed work.

### How Projections Work

- **After each task completes**, GSD calculates average cost per task
- **Multiplies by remaining tasks** in the roadmap
- **Adds completed cost** to estimate total

### Example

If you've completed 3 tasks at an average of $2.50 per task, and 7 tasks remain:

```
Completed: $7.50 (3 tasks)
Remaining: $17.50 (7 tasks estimated)
Projected total: $25.00
```

Projections improve as more work completes. Early projections are less accurate.

<Steps>
  <Step title="Sign up for OpenRouter">
    Get an API key at [openrouter.ai](https://openrouter.ai)

## Viewing Historical Costs

Cost data is stored in `.gsd/cost-ledger.json` (internal format). The dashboard aggregates this data.

To see historical costs:

```bash
/gsd status
```

Scroll through completed slices to see per-slice cost breakdowns.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Cost seems higher than expected">
    Check the dashboard for phase breakdown:
    
    ```bash
    /gsd status
    ```
    
    Planning phases are expensive if using Opus. Consider using Sonnet for simpler planning.
  </Accordion>
  
  <Accordion title="Budget ceiling not pausing auto mode">
    The ceiling is checked between tasks. If a task is very long, the ceiling may be exceeded before the check. Adjust timeouts in preferences to catch this earlier.
  </Accordion>
  
  <Accordion title="Projections wildly inaccurate">
    Projections improve as more work completes. Early estimates (after 1-2 tasks) are rough. Wait until 3-4 tasks complete for better accuracy.
  </Accordion>
</AccordionGroup>
