## Command Categories

### Workflow Commands

Core commands for executing work:

| Command | Description | Mode |
|---------|-------------|------|
| `/gsd` | Step mode â€” advance one unit at a time | Interactive |
| `/gsd next` | Explicit alias for step mode | Interactive |
| `/gsd auto` | Autonomous mode â€” complete milestones without intervention | Automated |
| `/gsd stop` | Stop auto mode gracefully | Control |

### Planning & Discussion

Commands for project planning and decision capture:

| Command | Description | Use Case |
|---------|-------------|----------|
| `/gsd discuss` | Discuss architecture and capture decisions | Gray areas, design choices |
| `/gsd queue` | Queue future milestones | Multi-milestone planning |

### Monitoring & Management

Commands for observability and control:

| Command | Description | Output |
|---------|-------------|--------|
| `/gsd status` | Progress dashboard | Overlay UI |
| `/gsd doctor` | Validate project integrity | Diagnostic report |
| `/gsd prefs` | Manage preferences | Opens editor |

### Migration & Setup

One-time or setup commands:

| Command | Description | When |
|---------|-------------|------|
| `/gsd migrate` | Convert v1 `.planning` to v2 `.gsd` | Legacy projects |
| `/worktree <name>` | Create parallel development workspace | Feature branches |
| `/voice` | Toggle speech-to-text | Hands-free mode |

## Command Workflow

### Starting a New Project

```bash
gsd
/gsd
```

With no `.gsd/` directory, GSD starts the **discussion flow** to capture your vision.

### Executing Work

**Step Mode** â€” Review each unit before proceeding:

```bash
/gsd next
# Complete one unit â†’ wizard shows what's next â†’ advance manually
```

**Auto Mode** â€” Walk away and let it build:

```bash
/gsd auto
# Research â†’ Plan â†’ Execute â†’ Complete â†’ Reassess â†’ Repeat
```

### Steering During Auto Mode

Run GSD in **two terminals** simultaneously:

**Terminal 1 â€” Let it build:**
```bash
gsd
/gsd auto
```

**Terminal 2 â€” Steer while it works:**
```bash
gsd
/gsd discuss    # capture architectural decisions
/gsd status     # check progress
/gsd queue      # queue the next milestone
```

Both terminals read and write the same `.gsd/` files. Your decisions are picked up automatically at the next phase boundary.

### Pausing and Resuming

Press `Escape` during auto mode to pause without losing state:

```bash
# During auto mode:
<Escape>  # Pauses, preserves context
# Interact with the agent, inspect output, review files
/gsd auto  # Resume from where it left off
```

## Command Completion

All commands support tab completion:

```bash
/gsd <tab>        # Shows: next, auto, stop, status, queue, discuss, prefs, doctor, migrate
/gsd prefs <tab>  # Shows: global, project, status
/worktree <tab>   # Shows existing worktree names + subcommands
```

## Command Design Principles

### One Command, Multiple Modes

`/gsd` is the universal entry point. It adapts based on project state:

- **No `.gsd/` directory** â†’ Discussion flow
- **Milestone exists, no roadmap** â†’ Research or discuss
- **Roadmap exists, slices pending** â†’ Plan the next slice or execute a task
- **Mid-task** â†’ Resume from where you left off

### Graceful Degradation

All commands handle edge cases:

- Running `/gsd status` with no milestones shows a helpful message
- Running `/gsd auto` on a complete milestone stops gracefully
- Running `/worktree merge` with no changes skips the merge

### Safety First

Destructive operations require confirmation:

- `/worktree remove` confirms before deletion
- `/gsd migrate` shows a preview before writing
- `/gsd doctor fix` only fixes safe issues automatically
