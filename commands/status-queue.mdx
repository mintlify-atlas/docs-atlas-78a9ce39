GSD provides real-time observability into your project's progress and cost. `/gsd status` shows the dashboard, `/gsd queue` plans future milestones.

## Commands

<ParamField path="/gsd status" type="command">
  Show the progress dashboard overlay
</ParamField>

<ParamField path="Ctrl+Alt+G" type="shortcut">
  Toggle dashboard overlay (same as `/gsd status`)
</ParamField>

<ParamField path="/gsd queue" type="command">
  Queue future milestones for multi-milestone planning
</ParamField>

## Status Dashboard

The dashboard shows:

- Current milestone, slice, and task
- Progress bars (slice and milestone completion)
- Active phase (research, plan, execute, complete)
- Cost and token usage (session and project totals)
- Completed units with timing and cost
- Auto mode state (active, paused, stopped)

## Status During Auto Mode

The dashboard updates **in real-time** while auto mode runs:

**Terminal 1 (auto mode):**

```bash
gsd
/gsd auto
```

**Terminal 2 (status):**

```bash
gsd
# Press Ctrl+Alt+G
# Dashboard shows live progress
```

The overlay refreshes as units complete. Press `Escape` or `Ctrl+Alt+G` again to close.

## Progress Widget (Auto Mode)

During auto mode, a compact widget shows in the footer:

```
🤖 auto — M001/S02/T01 — execute — 4m 23s — $1.24 — 45k tokens — claude-sonnet-4-6
```

**Fields:**

- **Mode** — `auto` (running), `paused` (escape pressed), or `step` (step mode)
- **Location** — Current milestone/slice/task
- **Phase** — `research`, `plan`, `execute`, `complete`, `reassess`
- **Elapsed** — Time since unit started
- **Cost** — Running session cost
- **Tokens** — Running session token count
- **Model** — Current model (may vary per phase if configured in preferences)

## Cost Projections

After the first slice completes, GSD estimates the total milestone cost:

```
📈 Projected milestone: ~$31.00 (2 slices remaining @ avg $6.22)
```

**Calculation:**

- **Average slice cost** = (sum of completed slice costs) / (number of completed slices)
- **Projected total** = (completed cost) + (remaining slices × average)

This helps you decide:

- Whether to continue or tune model selection
- If you're approaching your budget ceiling
- Whether to parallelize work with worktrees

## Queue: Future Milestones

`/gsd queue` plans **future milestones** without executing them immediately.

### Queue Flow

```bash
/gsd queue
```

**GSD asks:**

1. What's the next milestone about?
2. What does it depend on? (prior milestones, external blockers)
3. What's the rough scope? (number of slices)

**GSD writes:**

- `.gsd/QUEUE.md` — Ordered list of pending milestones
- `.gsd/M00X-CONTEXT.md` (stub) — High-level context for the milestone

### Example Queue Session

```bash
$ gsd
/gsd queue

💬 What's the next milestone?

> M002: User authentication and authorization

💬 What does it depend on?

  ◯ Nothing — can start immediately
  ◯ M001 must be complete
  ◯ External blocker (specify)

> M001 must be complete

💬 Rough scope estimate?

> 4-6 slices: signup, login, password reset, role-based access, session management, OAuth integration

✓ Queued: M002 — User authentication and authorization

Written to:
  .gsd/QUEUE.md
  .gsd/M002-CONTEXT.md (stub)

When M001 completes, auto mode will pick up M002 automatically.
```

### Queue During Auto Mode

Queue milestones while auto mode runs:

**Terminal 1 (auto mode):**

```bash
gsd
/gsd auto
# Building M001
```

**Terminal 2 (queue):**

```bash
gsd
/gsd queue
# Queue M002, M003, M004
```

When M001 completes, auto mode reads `QUEUE.md` and starts M002 automatically.

## Requirements Status

The dashboard shows **requirements tracking** if `REQUIREMENTS.md` exists:

```
📋 Requirements: 12 active · 8 validated · 3 deferred · 2 out of scope
```

**States:**

- **Active** — Not yet validated (still in progress)
- **Validated** — Implementation verified (tests pass, behavior confirmed)
- **Deferred** — Moved to a future milestone
- **Out of scope** — Explicitly excluded

See [Requirements](/concepts/milestones-slices-tasks) for details.

## Blockers

If the project has blockers, they're shown prominently:

```
⚠️  Blockers:
  • Waiting for API key from payment provider
  • Database migration pending (run `npm run migrate`)
```

Blockers come from:

- `CONTEXT.md` (depends_on: external dependencies)
- Task summaries (when a task can't complete without external input)
- Doctor diagnostics (integrity issues)

## Metrics File

All cost and token data is stored in `.gsd/metrics.jsonl` (gitignored):

```jsonl
{"timestamp":"2026-03-13T10:23:45Z","unit":"M001/S01","phase":"execute","model":"claude-sonnet-4-6","tokens":{"input":12450,"output":3890},"cost":0.18}
{"timestamp":"2026-03-13T10:25:08Z","unit":"M001/S01","phase":"complete","model":"claude-sonnet-4-6","tokens":{"input":8234,"output":2156},"cost":0.11}
```

This file is **append-only**. The dashboard aggregates it on-demand.

## Multi-Session Tracking

Metrics persist across sessions. If you stop and resume:

```bash
# Day 1
/gsd auto
# Complete S01, S02, then stop

# Day 2
/gsd auto
# Resume from S03
```

The dashboard shows **cumulative cost** from Day 1 + Day 2.
