# Installation

GSD is distributed as a global npm package. Install it once, use it in any project.

## Install GSD

<CodeGroup>
```bash npm
npm install -g gsd-pi
```

```bash yarn
yarn global add gsd-pi
```

```bash pnpm
pnpm add -g gsd-pi
```
</CodeGroup>

<Note>
The package name is `gsd-pi` on npm. The command you run is `gsd`.
</Note>

## System Requirements

GSD requires:

- **Node.js** ≥ 20.6.0 (22+ recommended)
- **Git** — initialized automatically if missing in your project
- **An LLM provider** — see [Choose Your Provider](#choose-your-provider) below

Optional (can be added later):
- Brave Search API key (web research)
- Tavily API key (web research — alternative to Brave)
- Google Gemini API key (web research via Gemini Search grounding)
- Context7 API key (library documentation lookup)
- Jina API key (clean web page extraction)

## First Launch: Setup Wizard

The first time you run `gsd`, you'll see a branded setup wizard that walks you through:

1. **LLM provider authentication** (OAuth or API key)
2. **Optional tool API keys** (web search, docs, etc.)

<Note>
Every step is skippable. Press **Enter** to skip any step. You can re-run the wizard anytime with `gsd config`.
</Note>

### What the Wizard Does

<Steps>
  <Step title="Display the GSD logo">
    Shows a branded welcome screen
  </Step>
  <Step title="Choose your LLM provider">
    Presents a list of 20+ supported providers with OAuth and API key options
  </Step>
  <Step title="Authenticate">
    Handles OAuth flows (opens browser, waits for callback) or collects API keys securely
  </Step>
  <Step title="Optional tool keys">
    Prompts for Brave Search, Context7, Jina, Slack, Discord — all optional
  </Step>
  <Step title="Summary and launch">
    Shows what was configured, then launches the GSD interactive session
  </Step>
</Steps>

```typescript
export async function runOnboarding(authStorage: AuthStorage): Promise<void> {

  // Step 3: Summary

}
```

### Credential Storage

All credentials are stored in `~/.gsd/auth.json` using the Pi SDK's `AuthStorage` format. This file is never committed to git.

If you already have Pi installed, GSD automatically imports your existing provider credentials — no need to re-authenticate.

### OAuth vs API Keys

**OAuth** (recommended for subscriptions):
- Opens browser for authentication
- Uses your existing subscription (Claude Max, GitHub Copilot)
- No API key needed
- Rate limits based on your plan

**API Keys** (recommended for pay-as-you-go):
- Paste your key when prompted
- Pay per token used
- No subscription required
- Full control over spending

## Re-run the Setup Wizard

You can re-run the setup wizard anytime:

```bash
gsd config
```

This reopens the full wizard experience — choose a different provider, add tool keys, or update existing credentials.

## Login from Inside GSD

If you skip the wizard on first launch, you can authenticate later from inside GSD:

```bash
gsd
/login
```

The `/login` command inside GSD opens the same provider selection flow.

### Switch Models

To change your default model after initial setup:

```bash
/model
```

This opens a model picker with all available models from your authenticated providers.

### Per-Phase Model Selection

Use different models for different phases in your preferences:

```bash
/gsd prefs
```

Example from the README (lines 296-303):

```yaml
---
version: 1
models:
  research: openrouter/deepseek/deepseek-r1
  planning: claude-opus-4-6
  execution: claude-sonnet-4-6
  completion: claude-sonnet-4-6
---
```

<Tip>
Use expensive models where quality matters (planning, complex execution) and cheaper/faster models where speed matters (research, simple completions).
</Tip>
