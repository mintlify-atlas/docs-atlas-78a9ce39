## Overview

`ExtensionAPI` is the primary interface for registering capabilities in your extension. It's passed to your default export function and provides methods to add tools, commands, event handlers, shortcuts, and more.

```typescript
export default function (pi: ExtensionAPI) {
  pi.registerTool({ ... });
  pi.registerCommand("mycommand", { ... });
  pi.on("before_agent_start", async (event, ctx) => { ... });
}
```

## Registering Tools

<ParamField path="registerTool" type="(tool: Tool) => void">
  Add a tool the LLM can invoke. Tools appear in the agent's tool palette.

  ```typescript
  pi.registerTool({
    name: "read_file",
    description: "Read a file from disk",
    inputSchema: {
      type: "object",
      properties: {
        path: { type: "string", description: "File path" }
      },
      required: ["path"]
    },
    execute: async (toolCallId, params, signal, onUpdate, ctx) => {
      const content = readFileSync(params.path, "utf-8");
      return { type: "text", text: content };
    }
  });
  ```

  ### Tool Interface
  - `name`: Tool identifier (lowercase, no spaces)
  - `description`: What the tool does (shown to LLM)
  - `inputSchema`: JSON Schema for parameters
  - `execute`: Async handler that returns a tool result
</ParamField>

## Registering Commands

<ParamField path="registerCommand" type="(name: string, config: CommandConfig) => void">
  Add a slash command users can invoke:

  ```typescript
  pi.registerCommand("gsd", {
    description: "GSD — Get Shit Done: /gsd next|auto|stop|status",
    
    getArgumentCompletions: (prefix: string) => {
      const subcommands = ["next", "auto", "stop", "status"];
      return subcommands
        .filter(cmd => cmd.startsWith(prefix))
        .map(cmd => ({ value: cmd, label: cmd }));
    },

    async handler(args: string, ctx: ExtensionCommandContext) {
      if (args === "status") {
        await ctx.ui.custom(/* render dashboard */);
      }
    }
  });
  ```

  ### CommandConfig
  - `description`: Help text shown in command palette
  - `getArgumentCompletions`: Optional autocomplete function
  - `handler`: Async function called when command is invoked

  <Tip>
    Use `getArgumentCompletions` for subcommand-style UX. See `/gsd` in the bundled extensions.
  </Tip>
</ParamField>

## Event Hooks

<ParamField path="on" type="(event: EventName, handler: EventHandler) => void">
  Subscribe to lifecycle events:

  ```typescript
  pi.on("before_agent_start", async (event, ctx) => {
    return { systemPrompt: `${event.systemPrompt}\n\nCustom instructions` };
  });

  pi.on("agent_end", async (event, ctx) => {
    // Session complete — save logs, emit metrics, etc.
  });

  pi.on("session_start", async (event, ctx) => {
    ctx.ui.notify("Extension loaded", "info");
  });
  ```

  See [Hooks](/api/hooks) for all event types and patterns.
</ParamField>

## Tool Overrides

Replace built-in tools with dynamic implementations:

```typescript
import { createBashTool } from "@mariozechner/pi-coding-agent";

const baseBash = createBashTool(process.cwd(), {
  spawnHook: (ctx) => ({ ...ctx, cwd: process.cwd() })
});

const dynamicBash = {
  ...baseBash,
  execute: async (toolCallId, params, signal, onUpdate, ctx) => {
    // Inject default timeout
    const paramsWithTimeout = {
      ...params,
      timeout: params.timeout ?? 120
    };
    return baseBash.execute(toolCallId, paramsWithTimeout, signal, onUpdate, ctx);
  }
};

pi.registerTool(dynamicBash as any);
```

From `gsd/index.ts:78-104`: GSD wraps the bash tool to inject a 120-second default timeout and dynamically resolve `process.cwd()` for worktree support.

## Custom Renderers

<ParamField path="registerToolRenderer" type="(name: string, renderer: ToolRenderer) => void">
  Customize how tool calls appear in the TUI:

  ```typescript
  pi.registerToolRenderer("my_tool", {
    render: (toolCall, result, theme) => {
      return `Custom rendering for ${toolCall.name}`;
    }
  });
  ```
</ParamField>
