# Build software while you sleep

GSD is an autonomous coding agent that decomposes work into milestones, slices, and tasks — then executes them with fresh context windows, automatic git branching, crash recovery, and cost tracking. Type `/gsd auto`, walk away, and come back to built software with clean git history.

## What makes GSD different

<CardGroup cols={2}>
  <Card title="Fresh context per task" icon="sparkles">
    Every task runs in a clean 200k-token window with pre-loaded context. No accumulated garbage.
  </Card>
  <Card title="Autonomous mode" icon="robot">
    `/gsd auto` runs research → plan → execute → verify → commit in a loop until the milestone is complete.
  </Card>
  <Card title="Branch-per-slice Git" icon="code-branch">
    Each slice gets its own branch with atomic commits, squash-merged to main as one clean commit.
  </Card>
  <Card title="Crash recovery" icon="life-ring">
    Lock files and session forensics let GSD resume exactly where it left off after any interruption.
  </Card>
  <Card title="Cost tracking" icon="dollar-sign">
    Per-phase model selection with token/cost ledgers and budget ceilings to control spending.
  </Card>
  <Card title="Stuck detection" icon="shield-check">
    Automatic retry with diagnostics when the agent produces unexpected output.
  </Card>
</CardGroup>

## Key features

<Steps>
  <Step title="Milestone/Slice/Task hierarchy">
    Work flows through demoable vertical slices, each decomposed into context-window-sized tasks with must-haves for verification.
  </Step>
  <Step title="State machine execution">
    Discuss → Research → Plan → Execute → Complete → Reassess. Each phase produces durable artifacts on disk.
  </Step>
  <Step title="Context engineering">
    Dispatch prompts include inlined task plans, prior summaries, dependency context, and decisions — no wasted tool calls on orientation.
  </Step>
  <Step title="13 bundled extensions">
    Browser tools, web search, Context7 docs, background shell, subagents, voice transcription, and more.
  </Step>
  <Step title="Multi-terminal workflow">
    Run auto mode in one terminal while steering from another. Both read the same `.gsd/` files.
  </Step>
</Steps>

## From prompt framework to agent application

The original GSD was a collection of markdown prompts for Claude Code. It worked, but had hard limits: no context control, no real automation, no crash recovery, no observability.

GSD v2 is a standalone CLI built on the Pi SDK with direct TypeScript access to the agent harness. That means GSD can **do** what v1 could only **ask** the LLM to do.
