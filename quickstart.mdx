## Install GSD

<CodeGroup>
```bash npm
npm install -g gsd-pi
```

```bash yarn
yarn global add gsd-pi
```

```bash pnpm
pnpm add -g gsd-pi
```
</CodeGroup>

<Note>Requires Node.js >= 20.6.0</Note>

## Log in to a Provider

First, choose your LLM provider:

<Steps>
  <Step title="Launch GSD">
    ```bash
    gsd
    ```
  </Step>
  
  <Step title="Run the login command">
    ```bash
    /login
    ```
  </Step>
  
  <Step title="Select your provider">
    Choose from 20+ providers: Anthropic, OpenAI, Google, OpenRouter, GitHub Copilot, and more.
    
    - If you have a **Claude Max** or **GitHub Copilot** subscription, the OAuth flow handles everything
    - Otherwise, paste your API key when prompted
  </Step>
  
  <Step title="Select a default model">
    GSD auto-selects a default model after login. To switch models later:
    
    ```bash
    /model
    ```
  </Step>
</Steps>

## Run Your First Command

Open a terminal in your project and run:

```bash
gsd
```

GSD opens an interactive agent session. From there, you have two ways to work:

<Tabs>
  <Tab title="Step Mode">
    Type `/gsd` and GSD executes one unit of work at a time, pausing between each with a wizard showing what completed and what's next.
    
    ```bash
    /gsd
    ```
    
    **What happens:**
    - No project yet? It starts the discussion flow
    - Roadmap exists? It plans or executes the next step
    - Same state machine as auto mode, but you stay in the loop
  </Tab>
  
  <Tab title="Auto Mode">
    Type `/gsd auto` and walk away. GSD researches, plans, executes, verifies, commits, and advances through every slice until the milestone is complete.
    
    ```bash
    /gsd auto
    ```
    
    **What happens:**
    - Fresh context window per task
    - Automatic git branching
    - Crash recovery
    - No babysitting required
  </Tab>
</Tabs>

## Two Terminals, One Project

The real workflow: run auto mode in one terminal, steer from another.

<CardGroup cols={2}>
  <Card title="Terminal 1: Let it build" icon="robot">
    ```bash
    gsd
    /gsd auto
    ```
  </Card>
  
  <Card title="Terminal 2: Steer while it works" icon="messages">
    ```bash
    gsd
    /gsd discuss    # talk through architecture decisions
    /gsd status     # check progress
    /gsd queue      # queue the next milestone
    ```
  </Card>
</CardGroup>

Both terminals read and write the same `.gsd/` files on disk. Your decisions in terminal 2 are picked up automatically at the next phase boundary - no need to stop auto mode.
